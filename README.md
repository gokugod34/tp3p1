# tp3p1
En mi proyecto, uso una Pila (Stack) cuando necesito que el último elemento agregado sea el primero en usarse, siguiendo la lógica LIFO (Last In – First Out). Esto funciona muy bien para el historial de cambios de notas y para las tareas pendientes del estudiante, donde lo más reciente tiene prioridad. Un ejemplo claro que lo representa en la vida real es cuando se usa "Ctrl + Z" en un editor de texto para deshacer, o cuando navego y quiero volver a la última página visitada.

Por otro lado, uso una Cola (Queue) cuando necesito respetar el orden en que los elementos ingresaron, siguiendo FIFO (First In – First Out). Esto encaja perfectamente para gestionar solicitudes o turnos, como cuando un profesor atiende estudiantes según el orden de llegada. En la vida real, un ejemplo sería una fila para pagar en un supermercado o una lista de tickets en un sistema de soporte.

Sobre la implementación, elegí trabajar con listas enlazadas porque son más flexibles que los arrays en este contexto. Una lista enlazada no necesita un tamaño fijo y permite agregar y quitar elementos sin tener que mover posiciones internas. En cambio, con arrays tendría que definir una capacidad y además desplazar elementos al eliminar o insertar en el medio. Las listas enlazadas se ajustan mejor a estructuras como pilas y colas, donde solo manejo los extremos (tope o frente) y no necesito acceder a elementos por índice directo.
